Other Cool Apps


By this point you should have set up your desktop and browser.   There are some applications that are worth mentioning.

  1. autofs This  system application automatically mounts NFS directories when needed, then dismounts them after a period of inactivity.  This tends to make cross-mounting more stable, especially when a machine goes down.  There are two directories where this happens.  Any reference to /home/hostname will mount /export/home on host hostname.  The other is more magical: /net/hostname will show all the exported directories from hostname.
  2. samba This application handles communication with Windows machines.  This host is configured to export /home/${HOSTNAME}/share as a Windows share that should be read/write available to Windows clients on your local network.  You can also use Samba to browse exported filesystems on Windows hosts.
  3. apache The http daemon will serve any files you put in your public_html subdirectory under http://localhost/~username/.  You can also put files in /var/www/localhost/htdocs.  It is possible to arrange an alias with the LANL hostmaster and have this host act as a virtual server for a project under http://myprojectname.lanl.gov where "myprojectname" is the name of an arbitrary project.
  4. thunderbird and evolution These two mail programs go far beyond terminal-based applications and are particularly good at handling those MS Word files and the like that people insist on sending.  Thunderbird is perhaps the more popular, although I use Evolution with the lab mail systems and like it.
  5. scipy The scipy scientific tools for python are installed.  These bind a lot of useful science and math libraries (LAPACK, gnuplot, image processing, and more) to the programming language python, and is particularly good for making publication-quality figures from data.
  6. nxclient and nxserver  These make secure remote X11 desktop connections to other machines.  The speed of the connections is surprisingly good.  Free clients are available for Windows  machines.  As a client, it's a great way to connect to a compute server in another room or building and have a full desktop available.  As a server (after some NX server configuration) it's a good way to connect to a Linux desktop from a Windows laptop.
  7. pdflatex Insert the command \usepackage[pdftex]{graphicx} into your Latex files and use the pdflatex command instead of latex to produce pdf files that are better for e-mailing and of equal quality for paper prints.  The one catch is that any included graphics must also be in PDF format.  If you are a gnuplot user, this is best done by producing eps files using the postscript driver and filtering them to PDF with the command epstopdf --compress --outfile=figureX.pdf figureX.eps.
  8. inkster You've probably heard of GIMP, which is a bitmap editor similar to PhotoShop, but you may not be familiar with this Scalable Vector Graphics (SVG) drawing editor.  Drawings produced by inkster may be included in web sites  and are good at any resolution.  They may also be used with LaTex in figures for publications.
  9. computational biology tools Emboss, Embassy, NCBI-tools, gromacs, and lots more.  Look in /usr/portage/sci-biology for the names of packages or use the esearch command to look for others.
  10. office tools OpenOffice and KOffice are both installed.  Each contains a full set of office applications (word processor, drawing, presentations, etc.)  I prefer OpenOffice at this time.
